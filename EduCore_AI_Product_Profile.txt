1. EXECUTIVE SUMMARY
────────────────────────────────────────────────────────────────────────────────

EduCore AI is a full-stack, AI-powered Learning Management System built
specifically for Indian universities. It replaces manual, time-consuming
academic workflows with intelligent automation — from AI-graded answer sheets
and question paper generation to a local private AI tutor running entirely
on-campus hardware.

Unlike cloud-only EdTech platforms, EduCore AI is self-hostable, data-private,
and deeply integrated with the real operational needs of university faculty,
students, administrators, and moderators.

Key differentiator: Every AI feature runs on the university's own GPU server
(NVIDIA RTX 3050 / CUDA), with no student data ever leaving the campus network.


────────────────────────────────────────────────────────────────────────────────
2. THE PROBLEM
────────────────────────────────────────────────────────────────────────────────

Universities across India face a common set of operational bottlenecks:

  [1] Manual Grading at Scale
      Evaluating hundreds of descriptive answer sheets takes weeks of faculty
      time, introduces inconsistency, and delays student feedback.

  [2] Question Paper Generation is Ad-Hoc
      Faculty manually draft questions from syllabus documents — a slow,
      error-prone process with no systematic coverage tracking.

  [3] Syllabus Coverage is Opaque
      Neither faculty nor students have a clear, interactive view of what has
      been covered, what is pending, and how it maps to exam topics.

  [4] Generic Learning Resources
      Existing LMS platforms provi\e content but no personalised guidance
      based on each student's enrolled course material.

  [5] Data Privacy Concerns with Cloud AI
      Sending student answer sheets or course documents to external cloud APIs
      violates institutional data governance and student privacy.

  [6] Fragmented Tooling
      Universities juggle separate tools for attendance, marks, LMS, career
      guidance, and communication — with no unified platform.


────────────────────────────────────────────────────────────────────────────────
3. THE SOLUTION — EDUCORE AI
────────────────────────────────────────────────────────────────────────────────

EduCore AI is a single unified platform with 8 core modules:

  MODULE 1 : AI Evaluation Engine
             Automatically grade descriptive answer sheets using configurable
             rubrics. Supports bulk evaluation via real-time WebSocket updates.

  MODULE 2 : Syllabus Engine
             Parse, analyse, and interactively explore syllabus documents with
             AI-generated topic breakdowns and coverage maps.

  MODULE 3 : Question Bank
             Repository of practise questions mapped to course topics with
             difficulty tagging and searchable filters.

  MODULE 4 : AI Tutor (On-Premise)
             Course-specific RAG chatbot powered by Ollama + Qwen2.5:7B.
             Runs entirely on campus GPU — zero cloud dependency.

  MODULE 5 : LMS Hub
             Full course management with enrolments, materials, assignments,
             and AI-evaluated submission grading.

  MODULE 6 : Career Counsellor
             AI-powered career guidance tailored to student profiles,
             skill assessments, and industry demand mapping.

  MODULE 7 : Moderation Workflow
             Structured question paper review and approval pipeline with
             admin assignment and moderator editor interface.

  MODULE 8 : Data Management
             Bulk import/export of students, faculty, courses, and marks
             with validation and audit trails.


────────────────────────────────────────────────────────────────────────────────
4. FLAGSHIP FEATURE — AI EVALUATION ENGINE
────────────────────────────────────────────────────────────────────────────────

The AI Evaluation Engine is EduCore's most impactful module.

HOW IT WORKS:
  - Faculty upload answer sheets (PDF / DOCX / image scans)
  - Define marking rubric: total marks, key concepts expected
  - AI reads and evaluates each answer against the rubric
  - Produces: marks awarded, justification, feedback per question
  - Bulk mode: evaluate entire class via real-time Socket.IO progress

BULK EVALUATION:
  - Faculty select multiple submissions and trigger batch evaluation
  - Live progress bar updates via WebSocket as each sheet is graded
  - Final marks and feedback downloadable as CSV / PDF report

EVALUATION HISTORY:
  - Full audit trail: who was graded, when, by which AI model, what rubric
  - Faculty can review, override, and resubmit any evaluation
  - Students see their feedback directly in Student Portal

IMPACT:
  - Reduces grading time from weeks to hours
  - Consistent, bias-free scoring across all submissions
  - Faculty freed to focus on teaching rather than marking


────────────────────────────────────────────────────────────────────────────────
5. AI TUTOR — ON-PREMISE PRIVATE AI
────────────────────────────────────────────────────────────────────────────────

EduCore AI includes a fully on-premise AI tutor powered by:
  - Ollama runtime   (local model serving)
  - Qwen2.5:7B model (7-billion parameter language model)
  - nomic-embed-text (local embedding model for semantic search)
  - RAG pipeline     (Retrieval-Augmented Generation)
  - NVIDIA RTX 3050 / CUDA (campus GPU server)

FEATURES:
  - Upload course materials: PDF, DOCX, PPTX, TXT
  - AI answers student questions citing exact source passages
  - Auto-generates quizzes (MCQ) from uploaded content
  - Auto-generates flashcards for revision
  - Full conversation history per student per course
  - Streaming responses via Server-Sent Events (SSE)

PRIVACY:
  - Zero data sent to any external server
  - All processing on campus hardware
  - Student queries and documents never leave the institution

────────────────────────────────────────────────────────────────────────────────
7. TECHNICAL ARCHITECTURE
────────────────────────────────────────────────────────────────────────────────

FRONTEND
  - React 18 + Vite 5
  - 24 views, CSS custom properties, RemixIcon 3.5, Inter font
  - Socket.IO client (real-time eval progress)
  - SSE reader (streaming AI chat)
  - Fully responsive (desktop + mobile sidebar)

BACKEND
  - Node.js 20 + Express 4
  - 14 route modules
  - Socket.IO server (bulk evaluation)
  - Server-Sent Events (AI chat streaming)
  - Multer 50MB file upload handling
  - RBAC enforced via x-user-id / x-user-role / x-user-name headers

DATABASE
  - PostgreSQL 15 (primary)
  - 20+ tables with audit timestamps
  - JSONB for vector embeddings (no pgvector extension required)
  - Partial unique indexes for data integrity
  - In-memory JSON fallback when PostgreSQL unavailable

AI STACK
  - Ollama runtime on campus GPU
  - Qwen2.5:7B (chat/evaluation)
  - nomic-embed-text (1536-dim embeddings)
  - JavaScript cosine similarity (vectorStore.js)
  - RAG pipeline: extract -> chunk -> embed -> retrieve -> generate

DEPLOYMENT
  - Self-hostable on any Linux/Windows server
  - Single npm start command
  - Environment config via .env
  - Designed for university on-premise infrastructure

8. ROLE-BASED ACCESS CONTROL (RBAC)
────────────────────────────────────────────────────────────────────────────────

EduCore AI has four distinct user roles with tailored access:

  ADMIN
    Full access to all modules: student & faculty management, course catalog,
    question bank, AI evaluation, syllabus engine, LMS, data management,
    moderation panel, AI tutor, notifications, feedback, career counsellor.

  FACULTY
    Access to: dashboard, courses, syllabus engine, AI evaluation, LMS,
    AI tutor, notifications, feedback board.

  STUDENT
    Access to: dashboard, enrolled courses, student portal (grades/profile),
    LMS, AI tutor, career counsellor, notifications, feedback.

  MODERATOR
    Access to: dashboard, assigned question paper moderation, notifications,
    feedback board.
