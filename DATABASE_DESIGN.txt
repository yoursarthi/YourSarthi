1. OVERVIEW
================================================================================

The ITM LMS serves three user roles — Admin, Faculty, and Student.
It manages the full academic lifecycle:

  • Student admissions, profiles, and login
  • Faculty and department management
  • Course catalog and course enrollment
  • Question bank with CO/Bloom's taxonomy tagging
  • AI-powered exam paper generation
  • AI evaluation of handwritten answer sheets (Google Gemini)
  • Examination hierarchy: Endsem/Midsem → Subject papers → Student results
  • Official marksheet generation with tamper-proof verification codes
  • LMS resources and announcements per course
  • Role-based access control (RBAC)
  • Student self-service portal with editable/locked profile fields

Primary database target: PostgreSQL 14+
Fallback (current): Flat JSON file (itm_lms_db.json) for development

3. STORAGE ARCHITECTURE
================================================================================

  [React Frontend]
       |
       | REST API + Socket.IO
       |
  [Express Server]
       |
       |── [PostgreSQL] ← Primary store for all academic records
       |        Tables: users, students, faculty, departments, programs,
       |                courses, course_enrollments, faculty_courses,
       |                questions, exam_papers, exam_questions,
       |                examinations, examination_papers,
       |                student_results, question_responses,
       |                marksheets, lms_resources, lms_announcements,
       |                eval_sessions, notifications,
       |                profile_change_requests
       |
       └── [JSON file] ← Development fallback only (itm_lms_db.json)
                         Used when DATABASE_URL env var is not set.
                         NOT suitable for production.

  Dual-write pattern (current):
    - Bulk evaluation writes to JSON always, PostgreSQL when available.
    - All other writes go directly to JSON until migration is complete.

4. TABLE DEFINITIONS
================================================================================

All IDs use VARCHAR(36) storing UUID v4 values unless stated otherwise.
All timestamps use TIMESTAMPTZ (timezone-aware).
Passwords must be stored as bcrypt hashes (min cost 10) — not plaintext.

────────────────────────────────────────────────────────────────────────────────
4.1  USERS
────────────────────────────────────────────────────────────────────────────────
Stores admin and faculty login credentials.
Students have their own table and use student_id as password.

  CREATE TABLE users (
    id            VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    email         VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,          -- bcrypt hash
    name          VARCHAR(255) NOT NULL,
    role          VARCHAR(20)  NOT NULL CHECK (role IN ('admin','faculty')),
    is_active     BOOLEAN      DEFAULT TRUE,
    last_login    TIMESTAMPTZ,
    created_at    TIMESTAMPTZ  DEFAULT NOW(),
    updated_at    TIMESTAMPTZ  DEFAULT NOW()
  );

  Notes:
  - Students do NOT appear in this table; they log in via the students table.
  - 'student' role is excluded from CHECK — only admin and faculty here.
  - last_login should be updated on every successful login.


────────────────────────────────────────────────────────────────────────────────
4.2  STUDENTS
────────────────────────────────────────────────────────────────────────────────
The primary student record. Also serves as the student login source
(password = student_id, e.g. ITM2024001).

  CREATE TABLE students (
    -- Admission / Identity (admin-locked)
    id                VARCHAR(36)  PRIMARY KEY,   -- e.g. ITM2024001
    enrollment_no     VARCHAR(100) UNIQUE,
    first_name        VARCHAR(100) NOT NULL,
    last_name         VARCHAR(100) NOT NULL,
    email             VARCHAR(255) UNIQUE NOT NULL,
    department_id     VARCHAR(36)  REFERENCES departments(id),
    program_id        VARCHAR(36)  REFERENCES programs(id),
    batch             VARCHAR(20)  NOT NULL,       -- e.g. 2024-2028
    category          VARCHAR(20)  DEFAULT 'General'
                        CHECK (category IN ('General','OBC','OBC-NCL','SC','ST','EWS')),
    father_name       VARCHAR(255) DEFAULT '',
    mother_name       VARCHAR(255) DEFAULT '',
    permanent_address TEXT         DEFAULT '',
    status            VARCHAR(20)  DEFAULT 'active'
                        CHECK (status IN ('active','inactive','pending','graduated')),

    -- Personal (student-editable)
    phone             VARCHAR(20)  DEFAULT '',
    whatsapp          VARCHAR(20)  DEFAULT '',
    gender            VARCHAR(30)  DEFAULT '',
    date_of_birth     DATE,
    nationality       VARCHAR(50)  DEFAULT 'Indian',
    religion          VARCHAR(50)  DEFAULT '',
    blood_group       VARCHAR(5)   DEFAULT '',
    medical_history   TEXT         DEFAULT '',
    local_address     TEXT         DEFAULT '',
    hobbies           TEXT         DEFAULT '',
    career_options    TEXT         DEFAULT '',
    achievements      TEXT         DEFAULT '',
    linkedin          VARCHAR(255) DEFAULT '',
    github            VARCHAR(255) DEFAULT '',

    created_at        TIMESTAMPTZ  DEFAULT NOW(),
    updated_at        TIMESTAMPTZ  DEFAULT NOW()
  );

  Notes:
  - Fields above the blank line are admin-only (locked from student edits).
  - Fields below can be updated freely by the student via the Student Portal.
  - department_id and program_id are FKs; department/program names are denormalised
    as strings in the JSON fallback.
  - The student logs in using: email + id (student_id) as password.
    Auth endpoint checks: students.email = ? AND students.id = ?


────────────────────────────────────────────────────────────────────────────────
4.3  PROFILE_CHANGE_REQUESTS
────────────────────────────────────────────────────────────────────────────────
When a student wants to change a locked field (e.g. name, department),
they submit a request. Admin approves/rejects.

  CREATE TABLE profile_change_requests (
    id            VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id    VARCHAR(36)  NOT NULL REFERENCES students(id) ON DELETE CASCADE,
    field_name    VARCHAR(100) NOT NULL,    -- e.g. 'first_name', 'department_id'
    old_value     TEXT         NOT NULL,
    requested_value TEXT       NOT NULL,
    reason        TEXT         DEFAULT '',
    status        VARCHAR(20)  DEFAULT 'pending'
                    CHECK (status IN ('pending','approved','rejected')),
    reviewed_by   VARCHAR(36)  REFERENCES users(id),
    reviewed_at   TIMESTAMPTZ,
    created_at    TIMESTAMPTZ  DEFAULT NOW()
  );


────────────────────────────────────────────────────────────────────────────────
4.4  DEPARTMENTS
────────────────────────────────────────────────────────────────────────────────

  CREATE TABLE departments (
    id          VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    name        VARCHAR(255) UNIQUE NOT NULL,
    code        VARCHAR(20)  UNIQUE NOT NULL,   -- e.g. CSE, ECE, MGT
    hod_id      VARCHAR(36)  REFERENCES faculty(id) ON DELETE SET NULL,
    created_at  TIMESTAMPTZ  DEFAULT NOW()
  );

  Current seed: CSE, MGT, ECE, ME, SCI, LAW

4.5  PROGRAMS
────────────────────────────────────────────────────────────────────────────────
Academic programs offered by the university.

  CREATE TABLE programs (
    id             VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    name           VARCHAR(255) UNIQUE NOT NULL,   -- e.g. B.Tech CSE
    short_name     VARCHAR(50),                    -- e.g. B.Tech
    department_id  VARCHAR(36)  REFERENCES departments(id),
    duration_years SMALLINT     DEFAULT 4,
    total_semesters SMALLINT    DEFAULT 8,
    is_active      BOOLEAN      DEFAULT TRUE,
    created_at     TIMESTAMPTZ  DEFAULT NOW()
  );

  Sample data:
    B.Tech CSE (8 sem), B.Tech ECE (8 sem), B.Tech ME (8 sem),
    BBA (6 sem), MBA (4 sem), BCA (6 sem), MCA (4 sem),
    B.Sc Hons (6 sem), BA LLB (10 sem), LLM (4 sem)


────────────────────────────────────────────────────────────────────────────────
4.6  FACULTY
────────────────────────────────────────────────────────────────────────────────

  CREATE TABLE faculty (
    id              VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    name            VARCHAR(255) NOT NULL,
    email           VARCHAR(255) UNIQUE NOT NULL,
    phone           VARCHAR(20)  DEFAULT '',
    department_id   VARCHAR(36)  REFERENCES departments(id) ON DELETE SET NULL,
    designation     VARCHAR(100) DEFAULT 'Assistant Professor'
                      CHECK (designation IN (
                        'Professor','Associate Professor','Assistant Professor',
                        'Lecturer','HOD','Visiting Faculty'
                      )),
    specialization  VARCHAR(255) DEFAULT '',
    qualification   VARCHAR(255) DEFAULT '',
    experience_years SMALLINT   DEFAULT 0,
    is_active       BOOLEAN      DEFAULT TRUE,
    created_at      TIMESTAMPTZ  DEFAULT NOW(),
    updated_at      TIMESTAMPTZ  DEFAULT NOW()
  );

  Notes:
  - faculty.email must match users.email for login to work.
    (users table holds the password; faculty table holds the profile.)
  - HOD of a department references faculty via departments.hod_id.


────────────────────────────────────────────────────────────────────────────────
4.7  COURSES
────────────────────────────────────────────────────────────────────────────────

  CREATE TABLE courses (
    id             VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    code           VARCHAR(20)  UNIQUE NOT NULL,    -- e.g. CSE301
    name           VARCHAR(255) NOT NULL,
    department_id  VARCHAR(36)  REFERENCES departments(id) ON DELETE SET NULL,
    credits        SMALLINT     NOT NULL DEFAULT 3,
    type           VARCHAR(20)  DEFAULT 'Core'
                     CHECK (type IN ('Core','Elective','Lab','Project','Audit')),
    max_students   SMALLINT     DEFAULT 60,
    semester       VARCHAR(10)  DEFAULT '1st',      -- 1st through 8th
    description    TEXT         DEFAULT '',
    is_active      BOOLEAN      DEFAULT TRUE,
    created_at     TIMESTAMPTZ  DEFAULT NOW(),
    updated_at     TIMESTAMPTZ  DEFAULT NOW()
  );

  Derived field: enrolled count = COUNT(course_enrollments WHERE course_id = ?)
  (Do not store enrolled count directly — compute from enrollments table.)


────────────────────────────────────────────────────────────────────────────────
4.8  FACULTY_COURSES  (Who teaches what)
────────────────────────────────────────────────────────────────────────────────
Maps which faculty member is assigned to teach which course, for a given
academic year and semester. Enables admin to view "courses run by each faculty."

  CREATE TABLE faculty_courses (
    id              VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    faculty_id      VARCHAR(36)  NOT NULL REFERENCES faculty(id) ON DELETE CASCADE,
    course_id       VARCHAR(36)  NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    academic_year   VARCHAR(20)  NOT NULL,   -- e.g. 2024-25
    semester_label  VARCHAR(10)  NOT NULL,   -- e.g. ODD, EVEN or specific sem
    is_active       BOOLEAN      DEFAULT TRUE,
    assigned_at     TIMESTAMPTZ  DEFAULT NOW(),
    UNIQUE (faculty_id, course_id, academic_year, semester_label)
  );


────────────────────────────────────────────────────────────────────────────────
4.9  COURSE_ENROLLMENTS  (Which students are in which course)
────────────────────────────────────────────────────────────────────────────────

  CREATE TABLE course_enrollments (
    id            VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    student_id    VARCHAR(36)  NOT NULL REFERENCES students(id) ON DELETE CASCADE,
    course_id     VARCHAR(36)  NOT NULL REFERENCES courses(id) ON DELETE CASCADE,
    academic_year VARCHAR(20)  NOT NULL,
    enrolled_at   TIMESTAMPTZ  DEFAULT NOW(),
    status        VARCHAR(20)  DEFAULT 'enrolled'
                    CHECK (status IN ('enrolled','dropped','completed')),
    UNIQUE (student_id, course_id, academic_year)
  );

  Notes:
  - This table is the source of truth for "enrolled" count on courses.
  - Students see only their enrolled courses in "My Courses".
  - Faculty see all students enrolled in their course.


────────────────────────────────────────────────────────────────────────────────
4.10  QUESTIONS  (Question Bank)
────────────────────────────────────────────────────────────────────────────────
Stores the question bank used for paper generation and AI evaluation.

  CREATE TABLE questions (
    id          VARCHAR(36)   PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id   VARCHAR(36)   REFERENCES courses(id) ON DELETE SET NULL,
    course_code VARCHAR(20)   NOT NULL,             -- denormalised for speed
    course_name VARCHAR(255)  DEFAULT '',
    type        VARCHAR(20)   NOT NULL
                  CHECK (type IN ('mcq','short','long','numerical','case-study')),
    topic       VARCHAR(255)  DEFAULT '',
    difficulty  VARCHAR(20)   DEFAULT 'medium'
                  CHECK (difficulty IN ('easy','medium','hard')),
    -- NBA/NAAC outcome mapping
    co          VARCHAR(20)   DEFAULT '',            -- Course Outcome e.g. CO1
    po          VARCHAR(20)   DEFAULT '',            -- Programme Outcome e.g. PO3
    blooms      VARCHAR(30)   DEFAULT ''
                  CHECK (blooms IN (
                    'remember','understand','apply',
                    'analyze','evaluate','create',''
                  )),
    -- Content
    text        TEXT          NOT NULL,
    options     JSONB         DEFAULT NULL,          -- {A:.., B:.., C:.., D:..}
    correct     VARCHAR(5)    DEFAULT NULL,          -- 'A','B','C','D' for MCQ
    marks       SMALLINT      DEFAULT 1,
    created_by  VARCHAR(36)   REFERENCES users(id) ON DELETE SET NULL,
    created_at  TIMESTAMPTZ   DEFAULT NOW()
  );


────────────────────────────────────────────────────────────────────────────────
4.11  EXAM_PAPERS
────────────────────────────────────────────────────────────────────────────────
A structured exam paper (question paper used for a specific subject in an exam).
Each exam paper belongs to one subject of one examination event.

  CREATE TABLE exam_papers (
    id               VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    title            VARCHAR(255) NOT NULL DEFAULT 'Untitled Exam',
    subject          VARCHAR(255) DEFAULT '',
    course_id        VARCHAR(36)  REFERENCES courses(id) ON DELETE SET NULL,
    program          VARCHAR(255) DEFAULT '',
    semester         VARCHAR(100) DEFAULT '',
    total_marks      INTEGER      DEFAULT 0,
    examination_id   VARCHAR(36)  REFERENCES examinations(id) ON DELETE SET NULL,
    created_by       VARCHAR(36)  REFERENCES users(id) ON DELETE SET NULL,
    created_at       TIMESTAMPTZ  DEFAULT NOW()
  );

  Notes:
  - examination_id links this paper to a parent Examination event
    (e.g. "End Semester 2024 — Sem 4").
  - One exam_paper = one subject's question paper for one exam event.


────────────────────────────────────────────────────────────────────────────────
4.12  EXAM_QUESTIONS
────────────────────────────────────────────────────────────────────────────────
Individual questions as they appear in an exam_paper (may differ from question
bank entries — marks, order, and rubric can be customised per paper).

  CREATE TABLE exam_questions (
    id             VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    paper_id       VARCHAR(36)  NOT NULL REFERENCES exam_papers(id) ON DELETE CASCADE,
    question_no    INTEGER      NOT NULL,
    question_text  TEXT         NOT NULL DEFAULT '',
    max_marks      INTEGER      NOT NULL DEFAULT 10,
    rubric         TEXT         DEFAULT '',
    sub_sections   JSONB        DEFAULT '[]',
    -- sub_sections structure:
    -- [{ label:"a", text:"...", maxMarks:5, rubric:"..." }, ...]
    created_at     TIMESTAMPTZ  DEFAULT NOW(),
    UNIQUE (paper_id, question_no)
  );


────────────────────────────────────────────────────────────────────────────────
4.13  EXAMINATIONS  (Parent examination event)
────────────────────────────────────────────────────────────────────────────────
The top-level examination event. All subject papers and results hang off this.

  CREATE TABLE examinations (
    id             VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    title          VARCHAR(255) NOT NULL,
    -- e.g. "End Semester Examination — B.Tech CSE Sem 4 (2024-25)"
    type           VARCHAR(30)  NOT NULL DEFAULT 'endsem'
                     CHECK (type IN ('endsem','midsem','internal','practical','viva')),
    semester       VARCHAR(20)  DEFAULT '',    -- e.g. 4th
    program_id     VARCHAR(36)  REFERENCES programs(id) ON DELETE SET NULL,
    academic_year  VARCHAR(20)  NOT NULL,      -- e.g. 2024-25
    start_date     DATE,
    end_date       DATE,
    status         VARCHAR(20)  DEFAULT 'scheduled'
                     CHECK (status IN ('scheduled','ongoing','evaluated','published')),
    -- 'published' = admin has released results to students
    published_at   TIMESTAMPTZ,
    published_by   VARCHAR(36)  REFERENCES users(id) ON DELETE SET NULL,
    created_by     VARCHAR(36)  REFERENCES users(id) ON DELETE SET NULL,
    created_at     TIMESTAMPTZ  DEFAULT NOW()
  );

  Lifecycle:
    scheduled → ongoing → evaluated → published
    Only when status = 'published' can students see their marks in My Portal.
    Only admin can set status = 'published'.


────────────────────────────────────────────────────────────────────────────────
4.14  EVAL_SESSIONS  (Bulk AI Evaluation Sessions)
────────────────────────────────────────────────────────────────────────────────
A session groups a batch upload of student answer sheets for the same subject.

  CREATE TABLE eval_sessions (
    id            VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    name          VARCHAR(255) DEFAULT 'Unnamed Session',
    subject       VARCHAR(255) DEFAULT '',
    max_marks     INTEGER      DEFAULT 100,
    paper_id      VARCHAR(36)  REFERENCES exam_papers(id) ON DELETE SET NULL,
    created_by    VARCHAR(36)  REFERENCES users(id) ON DELETE SET NULL,
    status        VARCHAR(20)  DEFAULT 'active'
                    CHECK (status IN ('active','completed','archived')),
    created_at    TIMESTAMPTZ  DEFAULT NOW()
  );


────────────────────────────────────────────────────────────────────────────────
4.15  STUDENT_RESULTS
────────────────────────────────────────────────────────────────────────────────
The AI-generated result for one student's answer sheet for one exam paper.

  CREATE TABLE student_results (
    id                   VARCHAR(36)   PRIMARY KEY DEFAULT gen_random_uuid(),
    paper_id             VARCHAR(36)   NOT NULL REFERENCES exam_papers(id) ON DELETE CASCADE,
    session_id           VARCHAR(36)   REFERENCES eval_sessions(id) ON DELETE SET NULL,
    student_id           VARCHAR(36)   REFERENCES students(id) ON DELETE SET NULL,
    student_name         VARCHAR(255)  DEFAULT '',   -- denormalised snapshot
    roll_no              VARCHAR(100)  DEFAULT '',
    enrollment_no        VARCHAR(100)  DEFAULT '',
    total_marks_obtained DECIMAL(7,2)  DEFAULT 0,
    max_marks            INTEGER       DEFAULT 0,
    percentage           DECIMAL(5,2)  DEFAULT 0,
    grade                VARCHAR(5)    DEFAULT 'F',
    transcription        TEXT          DEFAULT '',   -- AI-extracted answer text
    detailed_feedback    TEXT          DEFAULT '',
    strengths            JSONB         DEFAULT '[]',
    improvements         JSONB         DEFAULT '[]',
    comparison_notes     TEXT          DEFAULT '',
    teacher_note         TEXT          DEFAULT '',
    needs_review         BOOLEAN       DEFAULT FALSE,
    is_published         BOOLEAN       DEFAULT FALSE,  -- set TRUE when exam published
    created_at           TIMESTAMPTZ   DEFAULT NOW(),
    updated_at           TIMESTAMPTZ   DEFAULT NOW()
  );

  Notes:
  - student_id FK allows joining directly to the students table.
  - student_name/roll_no/enrollment_no are kept as snapshots in case the
    student record is later modified.
  - is_published mirrors the parent examination's published status.
    Student Portal only shows results WHERE is_published = TRUE.
  - grade scale: O(≥90), A+(≥80), A(≥70), B+(≥60), B(≥50), C(≥40), F(<40)

4.16  QUESTION_RESPONSES
────────────────────────────────────────────────────────────────────────────────
Per-question breakdown of marks within a student result.

  CREATE TABLE question_responses (

  );


────────────────────────────────────────────────────────────────────────────────
4.17  MARKSHEETS
────────────────────────────────────────────────────────────────────────────────
Official, digitally verifiable marksheet issued by admin after all subject
evaluations of an examination are complete.

  CREATE TABLE marksheets (
    id                VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    examination_id    VARCHAR(36)  REFERENCES examinations(id) ON DELETE CASCADE,
    student_id        VARCHAR(36)  REFERENCES students(id) ON DELETE CASCADE,
    student_name      VARCHAR(255) DEFAULT '',    -- snapshot at time of issue
    roll_no           VARCHAR(100) DEFAULT '',
    enrollment_no     VARCHAR(100) DEFAULT '',
    program           VARCHAR(255) DEFAULT '',
    semester          VARCHAR(100) DEFAULT '',
    academic_year     VARCHAR(20)  DEFAULT '',
    total_marks       INTEGER      DEFAULT 0,
    max_marks         INTEGER      DEFAULT 0,
    percentage        DECIMAL(5,2) DEFAULT 0,
    overall_grade     VARCHAR(5)   DEFAULT 'F',
    result_status     VARCHAR(10)  DEFAULT 'PASS'
                        CHECK (result_status IN ('PASS','FAIL','ATKT','ABSENT')),
    verification_code VARCHAR(100) UNIQUE NOT NULL,
    issued_by         VARCHAR(36)  REFERENCES users(id) ON DELETE SET NULL,
    issued_at         TIMESTAMPTZ  DEFAULT NOW(),
    remarks           TEXT         DEFAULT ''
  );

  Verification:
  - verification_code is a UUID or hash generated at issue time.
  - Public URL: /verify/:verification_code → shows student name, exam, marks.
  - Marksheet can only be issued by admin after examination.status = 'evaluated'.

  CREATE TABLE lms_resources (

  );


────────────────────────────────────────────────────────────────────────────────
4.20  LMS_ANNOUNCEMENTS
────────────────────────────────────────────────────────────────────────────────
Announcements posted per course. Hierarchy: admin can broadcast to all;
faculty broadcast to enrolled students of their course.

  CREATE TABLE lms_announcements (
    id            VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    course_id     VARCHAR(36)  REFERENCES courses(id) ON DELETE CASCADE,
    -- course_id NULL = university-wide broadcast (admin only)
    title         VARCHAR(255) NOT NULL,
    content       TEXT         DEFAULT '',
    author_id     VARCHAR(36)  REFERENCES users(id) ON DELETE SET NULL,
    author_name   VARCHAR(255) DEFAULT '',   -- snapshot
    audience      VARCHAR(20)  DEFAULT 'all'
                    CHECK (audience IN ('all','faculty','students')),
    is_pinned     BOOLEAN      DEFAULT FALSE,
    expires_at    TIMESTAMPTZ,
    created_at    TIMESTAMPTZ  DEFAULT NOW()
  );

  Audience logic:
    admin  → audience='all' with course_id=NULL  → visible to everyone
    admin  → audience='faculty'                  → visible to faculty only
    admin  → audience='students'                 → visible to students only
    faculty → course_id=<id>, audience='students' → students of that course only
    faculty → course_id=<id>, audience='all'      → all members of that course

4.21  NOTIFICATIONS
────────────────────────────────────────────────────────────────────────────────

  CREATE TABLE notifications (
    id            VARCHAR(36)  PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id       VARCHAR(36),   -- NULL = broadcast to all
    student_id    VARCHAR(36)  REFERENCES students(id) ON DELETE CASCADE,
    type          VARCHAR(30)  DEFAULT 'info'
                    CHECK (type IN ('info','success','warning','result','marksheet')),
    title         VARCHAR(255) NOT NULL,
    message       TEXT         DEFAULT '',
    link_page     VARCHAR(100) DEFAULT '',   -- e.g. 'student_portal'
    is_read       BOOLEAN      DEFAULT FALSE,
    created_at    TIMESTAMPTZ  DEFAULT NOW()
  );

10. KNOWN ISSUES IN CURRENT IMPLEMENTATION
================================================================================

  1. PLAINTEXT PASSWORDS
     users and students tables store passwords as plaintext in the JSON file.
     MUST be replaced with bcrypt hashes before any production use.

  2. NO COURSE ENROLLMENT TABLE
     The current schema has an 'enrolled' count on the courses table but no
     actual enrollment records. Students cannot be linked to specific courses.
     Add course_enrollments table and wire up "My Courses" for students.

  3. NO FACULTY-COURSE ASSIGNMENT
     Admin cannot currently see which courses a faculty member teaches.
     Add faculty_courses table.

  4. EXAMINATION HIERARCHY NOT IMPLEMENTED
     exam_papers have no examination_id yet. The examinations table and the
     parent-child structure are defined here but not yet in pgdb.js or db.js.
     student_results.is_published does not exist yet.

  5. MARKSHEET COVERS ONLY ONE SUBJECT
     The current marksheets table links to one result_id (one subject).
     marksheet_subjects (Section 4.18) must be added to support
     multi-subject marksheets per examination.

  6. PASS/FAIL BUG IN MARKSHEET
     Current code uses: data.grade >= 'C' (lexicographic string comparison).
     This incorrectly marks grade 'F' as PASS since 'F' > 'C' alphabetically.
     Fix: use numeric percentage threshold (percentage >= 40) or an explicit
     lookup: ['O','A+','A','B+','B','C'].includes(grade).

  7. STUDENT_RESULTS HAS NO STUDENT_ID FK
     student_results stores student_name and roll_no as text strings only.
     Add student_id VARCHAR(36) REFERENCES students(id) for proper joins.

  8. EVAL_SESSIONS NOT IN POSTGRESQL
     eval_sessions only exists in the JSON flat file.
     Should be migrated to PostgreSQL for proper session tracking.

  9. JSON FILE NOT SUITABLE FOR PRODUCTION
     The flat JSON file (itm_lms_db.json) has no concurrency control,
     no transactions, no referential integrity, and no indexing.
     It is only acceptable as a local development fallback.

  10. NO AUDIT LOG
      There is no record of who changed what and when.
      Add a generic audit_log table for compliance (NBA/NAAC requirement).

       SELECT sr.* FROM student_results sr
       JOIN exam_papers ep ON sr.paper_id = ep.id
       JOIN examinations e  ON ep.examination_id = e.id
       WHERE sr.student_id = $1
         AND sr.is_published = TRUE
       ORDER BY e.created_at DESC;

       UPDATE examinations SET status='published', published_at=NOW()
         WHERE id = $1;
       UPDATE student_results SET is_published=TRUE
         WHERE paper_id IN (
